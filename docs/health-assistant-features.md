# 건강상담도우미 에이전트 기능 설명서

> **버전**: v1.0  
> **작성일**: 2026-03-10  
> **작성자**: LEO

---

## 1. 서비스 개요

### 1.1 목적

건강상담도우미는 AI 기반 건강상담 에이전트로, 사용자의 증상을 체계적으로 파악하여 적절한 진료과를 안내하고, 주변 병의원 정보를 제공하는 서비스입니다.

### 1.2 대상 사용자

- 건강 이상 증상을 느끼지만 어떤 진료과를 방문해야 할지 모르는 일반인
- 병원 방문 전 사전 상담이 필요한 사용자
- 새로운 지역에서 적합한 병의원을 찾고자 하는 사용자

### 1.3 서비스 범위 및 면책

- 본 서비스는 **의료 행위가 아닌 건강 정보 안내** 목적으로 제공됩니다.
- AI의 판단은 참고용이며, 정확한 진단은 반드시 의료기관을 방문하여 받아야 합니다.
- 응급 상황(흉통, 호흡곤란, 의식 저하 등)은 즉시 119 또는 응급실 방문을 안내합니다.

## 2. 전체 이용 흐름

```
[서비스 진입] → [AI 문진 상담] → [진료과 추천] → [지역 입력] → [병의원 안내] → [병원 상세 정보]
```

| 단계 | 기능 | 핵심 산출물 |
|------|------|-------------|
| 1단계 | 서비스 진입 및 동의 | 이용동의 완료 |
| 2단계 | AI 문진 상담 | 증상 데이터 수집 |
| 3단계 | 진료과 추천 | 추천 진료과 + 근거 |
| 4단계 | 지역 입력 | 사용자 위치 정보 |
| 5단계 | 주변 병의원 안내 | 병의원 3개 리스트 |
| 6단계 | 병원 상세 정보 | 병원 정보 + 지도 + 웹페이지 |

## 3. 기능 상세

### 3.1 서비스 진입 및 동의

**기능 ID**: F-001

사용자가 서비스에 접속하면 간단한 서비스 소개와 이용약관 동의를 진행합니다.

- 서비스 소개: 건강상담도우미의 목적, 이용 방법, 면책 사항 안내
- 개인정보 수집 동의: 증상 정보, 위치 정보 수집에 대한 동의
- 의료 면책 고지: AI 상담은 의료 행위가 아님을 명시적으로 안내
- 응급 상황 안내: 응급 증상 시 119 또는 응급실 방문 권고

**동의 항목**:

- (필수) 서비스 이용약관 동의
- (필수) 개인정보 수집·이용 동의
- (선택) 위치 정보 활용 동의

---

### 3.2 AI 문진 상담

**기능 ID**: F-002

표준 문진표를 기반으로 AI가 체계적인 건강상담을 진행합니다.

#### 3.2.1 문진표 기반

건강검진 문진표 등 표준화된 문진 양식을 활용하여 신뢰성 있는 증상 수집을 수행합니다.

**참고 문진표 유형**:

- 국가건강검진 문진표 (국민건강보험공단)
- 일반 내과 초진 문진표
- 진료과별 특화 문진표 (안과, 정형외과, 피부과, 치과 등)

#### 3.2.2 상담 진행 방식

사용자 입력은 **자연어 대화형**과 **가이드형 선택지**를 병행합니다.

**Phase 1 — 초기 증상 파악**:

- "어디가 불편하신가요?" (자유 텍스트 입력)
- 입력된 증상을 분석하여 관련 신체 부위 및 증상 카테고리 분류

**Phase 2 — 상세 문진**:

- 증상 부위: 구체적인 통증/이상 부위 확인
- 증상 기간: 발생 시점 및 지속 기간
- 증상 강도: 1~10 척도로 통증/불편 정도 평가
- 동반 증상: 관련 동반 증상 유무 확인
- 과거 병력: 관련 질환 이력, 수술 이력
- 복용 약물: 현재 복용 중인 약물 정보
- 생활 습관: 흡연, 음주, 운동 등 (선택적)

**Phase 3 — 확인 및 보완**:

- 수집된 정보 요약 제시
- 추가 질문 또는 수정 사항 확인
- 최종 문진 데이터 확정

#### 3.2.3 문진 데이터 구조

```json
{
  "session_id": "string",
  "timestamp": "datetime",
  "symptoms": [
    {
      "description": "두통이 3일째 지속",
      "body_part": "머리",
      "category": "신경계",
      "duration": "3일",
      "severity": 6,
      "onset": "점진적"
    }
  ],
  "accompanying_symptoms": ["어지러움", "메스꺼움"],
  "medical_history": ["고혈압"],
  "current_medications": ["혈압약"],
  "vital_info": {
    "age_group": "40대",
    "gender": "남성"
  }
}
```

### 3.3 진료과 추천

**기능 ID**: F-003

문진 결과를 분석하여 방문해야 할 적절한 진료과를 추천합니다.

#### 3.3.1 증상-진료과 매핑

| 증상 카테고리 | 주요 증상 예시 | 추천 진료과 |
|--------------|---------------|-------------|
| 호흡기/전신 | 감기, 발열, 기침, 몸살 | 내과 |
| 소화기 | 복통, 소화불량, 구토 | 내과 (소화기내과) |
| 시력/안구 | 시력 저하, 눈 충혈, 안구 통증 | 안과 |
| 근골격계 | 관절통, 요통, 골절 의심 | 정형외과 |
| 피부 | 발진, 가려움, 여드름, 탈모 | 피부과 |
| 구강/치아 | 치통, 잇몸 출혈, 구내염 | 치과 |
| 이비인후 | 인후통, 귀 통증, 비염 | 이비인후과 |
| 비뇨기 | 배뇨 이상, 혈뇨 | 비뇨의학과 |
| 정신건강 | 불면, 우울, 불안 | 정신건강의학과 |
| 여성 건강 | 생리 불순, 하복부 통증 | 산부인과 |

#### 3.3.2 추천 결과 제시 형식

추천 결과는 다음 정보를 포함합니다:

- **추천 진료과**: 1순위 및 2순위 (해당 시)
- **추천 근거**: 어떤 증상이 해당 진료과 추천의 근거인지 설명
- **긴급도 표시**: 일반 / 조기 방문 권장 / 응급
- **참고 사항**: 복합 증상 시 우선 방문 순서 안내

**출력 예시**:

```
📋 문진 결과 분석

주요 증상: 두통(3일 지속), 어지러움, 메스꺼움
긴급도: ⚡ 조기 방문 권장

🏥 추천 진료과
  1순위: 신경과
    → 근거: 3일 이상 지속되는 두통 + 어지러움 동반
  2순위: 내과
    → 근거: 전신 증상 동반 가능성

💡 참고: 두통이 갑자기 심해지거나 시력 변화가 동반되면
   즉시 응급실을 방문해 주세요.
```

### 3.4 지역 입력 및 병의원 검색

**기능 ID**: F-004

사용자의 위치를 기반으로 추천 진료과에 해당하는 주변 병의원을 검색합니다.

#### 3.4.1 위치 입력 방식

- **GPS 자동 감지**: 사용자 단말의 위치 정보 활용 (위치 정보 동의 시)
- **주소 직접 입력**: 텍스트로 지역명 또는 주소 입력 (예: "강남역", "서울시 강남구")
- **지도 위 선택**: 지도에서 직접 위치를 선택

#### 3.4.2 병의원 검색 및 안내

추천 진료과에 해당하는 주변 병의원 **3개**를 선별하여 안내합니다.

**검색 기준**:

- 사용자 위치로부터의 거리 (가까운 순)
- 병원 평점 및 리뷰
- 현재 진료 가능 여부 (운영 시간 확인)

**리스트 표시 정보**:

- 병원명
- 진료과목
- 거리 (도보/차량 기준)
- 평점 (별점)
- 진료 가능 여부 (운영 중/마감)

**출력 예시**:

```
📍 강남역 주변 신경과 3곳

1. ○○신경과의원
   📌 강남역 3번 출구 도보 3분 (200m)
   ⭐ 4.5 (리뷰 128건)
   🕐 진료 중 (09:00~18:00)

2. △△내과신경과
   📌 강남역 5번 출구 도보 5분 (350m)
   ⭐ 4.3 (리뷰 89건)
   🕐 진료 중 (09:00~19:00)

3. □□종합의원
   📌 강남역 7번 출구 도보 7분 (500m)
   ⭐ 4.1 (리뷰 56건)
   🕐 점심시간 (13:00~14:00)
```

### 3.5 병원 상세 정보

**기능 ID**: F-005

사용자가 병원을 선택하면 해당 병원의 상세 정보를 표시합니다.

#### 3.5.1 기본 정보

- 병원명
- 진료 과목
- 주소 (도로명 주소)
- 전화번호 (원터치 전화 연결)
- 진료 시간 (요일별 상세)
- 점심시간
- 휴진일

#### 3.5.2 위치 정보 — 네이버 지도 연동

- 네이버 지도 임베드로 병원 위치 표시
- 현재 위치에서 병원까지 길찾기 기능 제공 (도보/대중교통/자가용)
- 네이버 지도 앱 열기 바로가기

#### 3.5.3 병원 웹페이지 연동

- 병원 공식 웹사이트 링크 제공
- 온라인 예약 페이지 바로가기 (지원 시)
- 병원 소개, 의료진 정보, 진료 안내 등 확인 가능

## 4. 에이전트 아키텍처

### 4.1 시스템 구성도

```
┌──────────────────────────────────────────────────┐
│                  사용자 인터페이스                   │
│            (Web / Mobile Chat UI)                 │
└──────────────────┬───────────────────────────────┘
                   │
┌──────────────────▼───────────────────────────────┐
│              오케스트레이터 에이전트                  │
│         (Supervisor / Router Agent)               │
│  - 사용자 의도 파악                                 │
│  - 에이전트 간 워크플로우 관리                       │
│  - 상태 관리 및 컨텍스트 유지                        │
└──┬──────────┬──────────┬──────────┬──────────────┘
   │          │          │          │
   ▼          ▼          ▼          ▼
┌──────┐ ┌──────┐ ┌──────┐ ┌──────────┐
│ 문진  │ │ 진단  │ │ 검색  │ │ 정보 제공 │
│에이전트│ │에이전트│ │에이전트│ │ 에이전트  │
└──┬───┘ └──┬───┘ └──┬───┘ └──┬───────┘
   │        │        │        │
   ▼        ▼        ▼        ▼
┌──────┐ ┌──────┐ ┌──────┐ ┌──────────┐
│문진표 │ │증상DB│ │지도API│ │병원정보DB │
│  KB  │ │매핑DB│ │위치API│ │  웹연동   │
└──────┘ └──────┘ └──────┘ └──────────┘
```

### 4.2 에이전트별 역할

| 에이전트 | 역할 | 사용 도구/리소스 |
|---------|------|-----------------|
| 오케스트레이터 | 전체 흐름 제어, 상태 관리 | LangGraph Supervisor |
| 문진 에이전트 | 표준 문진표 기반 증상 수집 | 문진표 KB, LLM |
| 진단 에이전트 | 증상 분석 및 진료과 매핑 | 증상-진료과 매핑 DB, LLM |
| 검색 에이전트 | 위치 기반 병의원 검색 | 지도 API, 위치 API |
| 정보 제공 에이전트 | 병원 상세 정보 조회·표시 | 병원 정보 DB, 네이버 지도 API |

### 4.3 기술 스택 (참고)

- **에이전트 프레임워크**: LangGraph (latest) (Supervisor 패턴)
- **LLM**: Claude / EXAONE 등 (on-premise 또는 API)
- **백엔드**: FastAPI (Python)
- **프론트엔드**: Next.js / React
- **지도 서비스**: 네이버 지도 API
- **병원 데이터**: 건강보험심사평가원 API, 공공데이터포털
- **Knowledge Base**: 표준 문진표, 증상-진료과 매핑 테이블

## 5. 데이터 흐름

### 5.1 전체 데이터 흐름

```
사용자 입력 (증상)
    │
    ▼
[문진 에이전트] ── 문진표 KB 참조 ──→ 구조화된 문진 데이터
    │
    ▼
[진단 에이전트] ── 증상-진료과 매핑 DB ──→ 추천 진료과 + 근거
    │
    ▼
사용자 입력 (위치)
    │
    ▼
[검색 에이전트] ── 지도/위치 API ──→ 주변 병의원 3개 리스트
    │
    ▼
사용자 선택 (병원)
    │
    ▼
[정보 제공 에이전트] ── 병원 정보 DB + 네이버 지도 ──→ 병원 상세 정보
    │
    ▼
상담 결과 요약 ──→ 저장/공유
```

### 5.2 외부 API 연동

| API | 용도 | 비고 |
|-----|------|------|
| 네이버 지도 API | 병원 위치 표시, 길찾기 | Maps, Geocoding, Directions |
| 건강보험심사평가원 API | 병의원 기본 정보 조회 | 공공데이터포털 제공 |
| 국민건강보험공단 | 표준 문진표 참조 | 건강검진 문진표 양식 |

## 6. 비기능 요구사항

### 6.1 보안 및 개인정보

- 사용자 건강 정보는 세션 종료 시 즉시 삭제 (비저장 원칙)
- 모든 통신은 HTTPS/TLS 암호화 적용
- 개인정보 처리방침 명시 및 동의 절차 준수
- 민감 의료 정보에 대한 PII 처리 (마스킹/익명화)

### 6.2 사용성

- 문진 응답 소요 시간: 3분 이내 목표
- 전체 상담 소요 시간: 5분 이내 목표
- 모바일 최적화 (반응형 UI)
- 고령 사용자 고려: 큰 글씨, 음성 입력 지원 (향후)

### 6.3 신뢰성

- AI 추천에 대한 근거 투명성 확보
- 의료 면책 고지 상시 노출
- 응급 상황 감지 시 즉각 응급 안내 전환
- 판단 불가 시 "의료기관 직접 방문" 안내
