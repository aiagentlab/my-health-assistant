## 1. 개요

### 1.1 목적

본 문서는 건강상담도우미 에이전트의 사용자 여정을 6단계로 정의하고, 각 단계별 사용자 행동, 감정, 터치포인트, Pain Point, 기회 요소를 체계적으로 정리합니다.

### 1.2 대상 사용자

- **주 사용자**: 건강 이상 증상을 느끼지만 어떤 진료과를 방문해야 할지 모르는 일반인
- **페르소나**: 바쁜 직장인(30대), 건강 걱정 많은 중장년층(50대), 타지 방문자(20대)

### 1.3 전체 여정 흐름

```
[1.진입] → [2.문진] → [3.진단 안내] → [4.병원 탐색] → [5.병원 상세] → [6.행동]
```

## 3. 단계별 상세 Journey Map

### 3.1 1단계 — 진입 (Entry)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 서비스 인지 및 이용 시작 |
| **터치포인트** | 앱/웹 접속, 서비스 소개 확인 |
| **관련 화면** | S-01 온보딩/동의 |

**사용자 행동**:

- 건강상담도우미 서비스에 진입
- 서비스 소개 및 이용 방법 확인
- 이용약관 및 개인정보 동의 진행

**사용자 생각**:

- "어디가 아픈데 어디로 가야 하지?"
- "간단하게 상담받을 수 있을까?"

**Pain Points**:

- 의료 정보에 대한 불확실성
- 어떤 병과를 방문해야 할지 모름
- AI 상담에 대한 신뢰도 우려

**기회 요소**:

- 친근한 톤의 온보딩으로 심리적 장벽 낮추기
- 간단한 서비스 플로우(3단계) 안내 제공
- 의료 면책 고지를 명확히 하되 불안감을 최소화

**주요 화면**:

- 스플래시/랜딩 페이지
- 이용동의 화면

---

### 3.2 2단계 — 문진 (Health Screening)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 사용자 증상을 체계적으로 수집 |
| **터치포인트** | AI 챗봇 인터페이스, 표준 문진표 |
| **관련 화면** | S-02 AI 문진 상담 |

**사용자 행동**:

- 증상 입력 (자유 텍스트 또는 선택형)
- 표준 문진표 기반 질문에 응답
- 증상 부위, 기간, 강도 등 상세 정보 입력
- 동반 증상, 과거 병력, 복용 약물 정보 제공 (선택)

**사용자 생각**:

- "이 정도 정보면 충분할까?"
- "내 증상을 정확히 설명할 수 있을까?"
- "언제쯤 결과가 나오지?"

**Pain Points**:

- 의학 용어 이해 어려움
- 증상 표현의 모호함 (어떻게 설명해야 할지 모름)
- 문진이 너무 길면 이탈 위험
- 자신의 증상이 심각한 것인지 불안

**기회 요소**:

- 건강검진 문진표 등 표준화된 문진 활용으로 신뢰성 확보
- 자연어 입력 + 가이드형 선택지 병행으로 입력 부담 감소
- 단계별 프로그레스바로 진행률 표시 (이탈 방지)
- 쉬운 용어 사용 및 필요 시 용어 설명 팝업 제공
- 문진 3분 이내 완료 목표로 피로감 최소화

**주요 화면**:

- 증상 입력 채팅 화면
- 가이드형 문진표 UI (선택지 칩, 기간 그리드, 강도 슬라이더)

---

### 3.3 3단계 — 진단 안내 (Department Recommendation)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 적합한 진료과를 추천하고 근거를 제시 |
| **터치포인트** | AI 분석 결과 화면 |
| **관련 화면** | S-03 진료과 추천 결과 |

**사용자 행동**:

- 추천 진료과 확인 (1순위, 2순위)
- 증상-진료과 매핑 근거 확인
- 긴급도 확인 (일반 / 조기 방문 권장 / 응급)
- 필요 시 추가 질문 또는 재상담 요청

**사용자 생각**:

- "이 결과가 맞는 건가?"
- "왜 이 진료과를 추천하는 거지?"
- "내 증상이 심각한 건 아닌지..."

**Pain Points**:

- AI 판단에 대한 신뢰도 우려
- 여러 증상이 복합적일 때 어떤 진료과가 맞는지 혼란
- 추천 근거가 불충분하면 불안감 증가

**기회 요소**:

- 추천 근거를 투명하게 제시하여 신뢰 확보
- 의료 면책 고지를 명확히 표시하되 불안감 유발 최소화
- 복수 진료과 추천 시 우선순위와 방문 순서 안내
- 긴급도에 따른 차별화된 안내 (응급 시 119 즉시 연결)
- 재상담 옵션 제공으로 사용자 주도권 보장

**주요 화면**:

- 진료과 추천 결과 카드 (1순위 강조, 2순위 보조)
- 추천 근거 설명 패널
- 긴급도 배지 및 응급 경고 배너

---

### 3.4 4단계 — 병원 탐색 (Hospital Search)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 주변 적합한 병의원을 찾아 비교 |
| **터치포인트** | 위치 입력 UI, 병원 목록 화면 |
| **관련 화면** | S-04 위치 입력 + 병원 검색 |

**사용자 행동**:

- 지역/위치 입력 (주소 직접 입력 또는 GPS 자동 감지)
- 주변 병의원 3개 리스트 확인
- 거리, 평점, 진료시간 비교
- 지도에서 병원 위치 확인

**사용자 생각**:

- "가까운 곳이 어디지?"
- "평이 좋은 병원은 어디일까?"
- "지금 진료 가능한 곳이 있을까?"

**Pain Points**:

- 너무 많은 정보로 인한 선택 장애
- 실시간 진료 가능 여부 불확실
- 병원 리뷰/평점의 신뢰성 의문

**기회 요소**:

- 핵심 정보(거리, 평점, 대기시간) 우선 표시로 의사결정 지원
- 지도 기반 직관적 UI 제공
- Top 3 추천으로 선택지 제한 (선택 피로 방지)
- 현재 진료 가능 여부(운영 중/점심/종료) 실시간 표시
- 야간/휴일 진료 가능 병원 우선 노출

**주요 화면**:

- 위치 입력 화면 (텍스트 + GPS 버튼)
- 병원 리스트 + 미니맵 통합 화면

---

### 3.5 5단계 — 병원 상세 (Hospital Detail)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 선택한 병원의 상세 정보를 확인하고 방문 결정 |
| **터치포인트** | 병원 상세 정보 페이지, 네이버 지도 연동 |
| **관련 화면** | S-05 병원 상세 정보 |

**사용자 행동**:

- 병원 선택
- 기본 정보 확인 (주소, 전화번호, 진료시간, 휴진일)
- 지도에서 위치 확인 및 길찾기 (도보/대중교통/자가용)
- 병원 웹페이지 방문 (의료진 정보, 진료 안내 등)
- 전화 연결 또는 예약

**사용자 생각**:

- "이 병원이 괜찮을까?"
- "지금 진료 가능할까?"
- "어떻게 가야 하지?"

**Pain Points**:

- 병원 정보의 최신성 불확실 (진료시간 변경 등)
- 예약 연동 부재 시 불편 (전화 예약만 가능)
- 병원 웹페이지가 없거나 정보가 부실한 경우

**기회 요소**:

- 실시간 운영정보 연동으로 최신성 확보
- 네이버 지도 길찾기 바로가기로 즉각적인 행동 유도
- 전화 원터치 연결 (tel: 프로토콜)
- 병원 웹페이지 링크로 추가 정보 탐색 지원
- 이동수단별(도보/대중교통/자가용) 예상 시간 표시

**주요 화면**:

- 병원 상세 정보 카드
- 네이버 지도 임베드 + 길찾기 버튼
- 전화/홈페이지 CTA 버튼

### 3.6 6단계 — 행동 (Action & Follow-up)

| 항목 | 내용 |
|------|------|
| **단계 목표** | 상담 결과를 활용하여 실제 행동으로 연결 |
| **터치포인트** | 요약 화면, 알림/리마인더 |
| **관련 화면** | S-06 상담 요약 |

**사용자 행동**:

- 상담 결과 요약 확인 (증상 + 추천 진료과 + 선택 병원)
- 결과 저장 또는 공유 (카카오톡, PDF)
- 병원 방문 또는 전화 예약
- 만족도 피드백 제출
- 이후 재상담 또는 서비스 종료

**사용자 생각**:

- "다음에 또 이용해야지"
- "결과를 저장해두고 나중에 보고 싶다"
- "가족에게 공유해서 같이 얘기해봐야지"

**Pain Points**:

- 상담 이력 관리가 되지 않으면 재방문 시 처음부터 다시 해야 함
- 후속 조치(병원 방문)를 잊어버림
- 결과를 다른 사람과 공유하기 불편

**기회 요소**:

- 상담 결과 PDF/카카오톡 공유로 가족·지인과 상의 지원
- 방문 리마인더 알림 (푸시 알림, 향후)
- 재방문 시 이전 상담 이력 연계 (향후)
- 만족도 피드백 수집으로 서비스 개선 루프 구축
- 새 상담 시작 진입점 제공으로 재사용 유도

**주요 화면**:

- 상담 요약 카드 (증상 + 진료과 + 병원 통합)
- 저장/공유 버튼 영역
- 만족도 피드백 UI

## 5. 핵심 개선 기회 요약

### 5.1 Quick Wins (즉시 적용 가능)

- 문진 프로그레스바 표시로 이탈률 감소
- 추천 근거 투명 공개로 신뢰도 향상
- Top 3 병원 제한으로 선택 피로 방지
- 전화 원터치 연결로 행동 전환율 향상

### 5.2 Medium-term (v1.1~v1.2)

- 상담 이력 관리로 재방문 경험 개선
- 카카오톡/PDF 공유로 결과 활용도 증대
- 야간/휴일 진료 가능 병원 우선 노출

### 5.3 Long-term (v2.0+)

- 음성 입력(STT) 지원으로 고령자 접근성 강화
- 병원 온라인 예약 연동으로 end-to-end 경험 완성
- 방문 리마인더 푸시 알림
- 이전 상담 이력 기반 맞춤형 문진

```
────────────────── Frontstage (사용자에게 보이는 영역) ──────────────────

사용자    [접속]   [증상입력]   [결과확인]   [위치입력]   [병원선택]   [저장/공유]
           │         │           │           │           │           │
UI        S-01      S-02        S-03        S-04        S-05        S-06

═══════════════════════════════ Line of Visibility ═══════════════════════

────────────────── Backstage (사용자에게 보이지 않는 영역) ──────────────

에이전트    -     문진Agent   진단Agent   검색Agent   정보Agent     -
           │         │           │           │           │           │
시스템    동의처리  문진표KB    매핑DB     지도API     병원DB     결과저장

                   응급감지   가중치계산   반경조정    웹연동     카카오API

═══════════════════════════════ Line of Internal Interaction ═════════════

외부시스템    -       -       증상DB     네이버API    심평원API     -
                            매핑테이블   위치API     병원웹사이트
```
