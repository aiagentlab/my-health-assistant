## 1. 문서 개요

### 1.1 목적

본 문서는 건강상담도우미 에이전트의 전체 화면 구성, UI 컴포넌트 명세, 화면 간 전환 흐름, 인터랙션 정의를 기술합니다. PRD의 기능 요구사항(FR-001~FR-013)과 User Journey Map의 6단계를 기반으로 설계되었습니다.

### 1.2 화면 목록

| 화면 ID | 화면명 | 관련 FR | Journey 단계 |
|---------|--------|---------|-------------|
| S-01 | 온보딩/동의 | FR-001 | 1. 진입 |
| S-02 | AI 문진 상담 | FR-002, FR-008 | 2. 문진 |
| S-03 | 진료과 추천 결과 | FR-003 | 3. 진단 안내 |
| S-04 | 위치 입력 + 병원 검색 | FR-004 | 4. 병원 탐색 |
| S-05 | 병원 상세 정보 | FR-005, FR-006, FR-007 | 5. 병원 상세 |
| S-06 | 상담 요약 | FR-009, FR-010 | 6. 행동 |

### 1.3 설계 원칙

- **신뢰감**: 의료 서비스에 적합한 안정적이고 전문적인 비주얼 톤
- **접근성**: WCAG 2.1 AA 준수, 큰 터치 영역(최소 44×44px), 고대비
- **효율성**: 최소 탭으로 목표 달성, 불필요한 단계 제거
- **안심감**: 면책 고지, 응급 안내 등 안전장치 상시 노출
